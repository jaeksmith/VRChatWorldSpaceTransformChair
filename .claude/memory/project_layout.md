**Why the `Assets/Local/` split:** keeps the user's authored content cleanly separate from imported 3rd-party SDK/UdonSharp/XR assets. Anything outside `Local/` is treated as a vendored dependency — don't hand-modify it.

**How to apply:**
- All new scripts, prefabs, scenes, materials, etc. go under `Assets/Local/...`.
- Generated artifacts (atlases, baked data, etc.) go under `Assets/Local/Generated/` and should be regenerable from Editor menus.
- For non-Unity tooling (asset pipelines, build scripts), put them under the project root outside `Unity/` (e.g. `Tools/` if needed).
